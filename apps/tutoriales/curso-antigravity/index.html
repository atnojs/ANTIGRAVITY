Curso Completo de Google Antigravity

De Copiloto a Agente: Multi-Agentes, Depuración y Contexto Avanzado.

Este curso está dirigido a desarrolladores que buscan dar el salto de usar la IA como un simple asistente de autocompletado a integrarla como agentes autónomos en su flujo de trabajo diario.

Ver Tutorial en YouTube

1. El cambio de paradigma

Google Antigravity no es solo un editor más; es un fork de Visual Studio Code diseñado específicamente para trabajar con agentes de IA. El cambio fundamental reside en pasar de ser "escritores de código" a convertirnos en "arquitectos de software" que supervisan el trabajo de agentes inteligentes.

Tip: Antigravity permite decidir el nivel de autonomía del agente: desde un simple asistente hasta un planificador autónomo.

2. Acceso e Interfaz

Para comenzar, debes descargar la versión correspondiente a tu sistema operativo en antigravity.google. La interfaz es casi idéntica a VS Code, lo que facilita la transición.

  1. Descarga e instala la aplicación para Mac, Windows o Linux.
  2. Vincula tu cuenta de Google (Free, Pro o Ultra).
  3. Explora el panel derecho (Agent Panel) y el Open Agent Manager.
Importante: Antigravity no usa API Keys externas; funciona exclusivamente vinculado a tu cuenta de Google.

3. Modos de Trabajo: Fast vs Planning

Antigravity permite alternar entre dos motores de razonamiento según la necesidad:

  • Fast Mode (Velocidad): Respuestas instantáneas para preguntas rápidas o refactorizaciones de una sola función. Es el equivalente a los LLMs tradicionales.
  • Planning Mode (Estructura): Vital para tareas complejas. El agente se detiene a pensar antes de actuar, generando un Artefacto (un contrato o plano visual) que detalla pasos lógicos, estructura de archivos y comandos de terminal.
Atajos de Teclado:
  • Ctrl + I: Modo Inline (edición directa y contextual).
  • Ctrl + L: Abrir el Panel de Agente lateral.

4. Práctica: Construcción y Refactorización

4.1 Creación de App Flask (Modo Planning)

Delegamos el proceso completo al agente para entender el flujo de trabajo agéntico:

  1. Escribe el prompt: "Crea una aplicación Flask que muestre la fecha actual, instala dependencias, arranca el servidor y abre el navegador."
  2. Observa la generación del Artefacto. Hasta que no firmes el "contrato" (Approve Plan), el agente no empezará a trabajar.

4.2 Refactorización Inline (Ctrl + I)

No reescribas código a mano. Usa el comando Inline directamente sobre el editor:

  • Evolución: "Convierte esta calculadora JS a TypeScript con interfaces y manejo de errores."
  • Arquitectura: "Refactoriza esto para usar una estructura MVC (Model-View-Controller) limpia."
  • Documentación: "Genera documentación JSDoc explicativa para estos parámetros."

5. El Agente con "Ojos"

La Browser Surface permite al agente interactuar con el navegador Chrome integrado. No solo lanza la web, sino que puede "ver" los elementos y probarlos como un usuario real.

Tip: Puedes pedirle al agente: "Abre la app, añade una tarea y verifica que el botón de borrar funciona."

6. Debugging Autónomo: Sherlock Holmes Digital

Antigravity no solo arregla errores de sintaxis, sino fallos de lógica profundos. No necesitas copiar y pegar el error; el agente tiene integración contextual completa.

  1. Mención de Terminal: Usa @terminal y pide: "Investiga el último fallo y arréglalo."
  2. Investigación: El agente inyectará logs temporales para verificar tipos de datos y formular una hipótesis.
  3. Resolución: Arreglará el fallo (ej. un String colado en una suma) y limpiará sus rastros, eliminando los logs de depuración creados.
Bucle de Seguridad: Si el agente entra en un bucle de fallos, se detendrá. Puedes ordenarle: "Sigue intentando hasta que funcione" directamente en el chat.

7. Seguridad y Supervisión

La seguridad es prioritaria mediante el uso de Sandboxes (Jaulas lógicas). El agente solo tiene acceso a la carpeta del proyecto actual.

Advertencia: Configura siempre Review Policy en "Request Review" para Supervisar cualquier cambio crítico en tus archivos o estrategia.

8. Orquestación Multi-Agente: Manager View

Pasa de una relación lineal a ser el director de una orquesta de IAs. Puedes coordinar expertos trabajando en paralelo sobre un mismo sistema complejo.

Lógica de Compartición (Escenario vs Mente):
  • Comparten escenario: Ambos ven los archivos del proyecto y cambios en el disco.
  • No se leen la mente: El agente 2 no tiene acceso al chat ni a los pensamientos del agente 1.
Como director, debes actuar como puente de contexto: "Tu compañero ha creado el archivo X, ahora tú haz Y."

9. Gestión de Workspaces

Puedes supervisar múltiples proyectos (Workspaces) desde una única interfaz simplificada. Cada workspace es un entorno aislado, asegurando que un proyecto no interfiera con otro.

10. Contexto Avanzado

Inyecta conocimiento específico de tu empresa (reglas de negocio, guías de estilo) para que la IA deje de ser genérica.

Tip: Usa menciones con @ para forzar a la IA a consultar documentos específicos antes de generar código.

11. Estrategia Markdown

El formato .md es la "Constitución" de tu proyecto. Es texto plano que los modelos de razonamiento entienden a la perfección para seguir reglas de diseño o arquitectura.

12. Demo Final: Invocación "@" de Contexto

Ignora el conocimiento generalista de la IA y oblígala a seguir tus reglas corporativas mediante el uso de la araba (@).

Caso de Uso: Si vinculas un archivo guiaestilo.md, el agente consultará tus fuentes exactas (ej. colores naranja FF5733 y bordes de 12px) antes de generar cualquier componente, actuando como un consultor diligente que conoce tu cultura de empresa.

13. Conclusión

Antigravity no viene a sustituir al desarrollador, sino a eliminar la parte monótona del trabajo. El futuro pertenece al Desarrollador Híbrido: dirección estratégica humana potenciada por ejecución incansable de la máquina.

¿Qué hacer después de este curso?

  • Proyecto 1: Crea un dashboard interactivo que consuma una API pública usando el Modo Planning.
  • Proyecto 2: Implementa una suite de tests end-to-end automatizada usando la Browser Surface.
  • Proyecto 3: Define tu propia "Constitución de Código" en Markdown y deja que los agentes refactoricen un proyecto antiguo basado en ella.
