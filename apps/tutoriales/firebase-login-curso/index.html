Gemini Drive: Backend con Firebase

De una app estática a una nube personal persistente.

Aprende a dotar a tus aplicaciones de "memoria" y "seguridad" mediante la integración de Firebase Authentication, Firestore y Cloud Storage. Convierte tu front-end en un Gemini Drive real.

Ver Tutorial en YouTube

1. El poder del Backend No-Code

Firebase es la herramienta definitiva para desarrolladores que quieren centrarse en el producto y no en la infraestructura. Con una sola herramienta gratuita, gestionamos usuarios, base de datos y almacenamiento masivo de archivos.

2. La Lógica del Casillero

Imagina tu backend como un sistema de casilleros:

  • Authentication es el sistema de llaves (quién puede entrar).
  • Firestore es el inventario (una lista de qué hay en cada casillero).
  • Storage es el casillero físico (donde guardas los objetos pesados/archivos).

3. Setup en la Consola

Crea un proyecto llamado Gemini Drive en Firebase Console. No es necesario activar Google Analytics para este tutorial.

  1. Entra en Firebase Console.
  2. Crea un proyecto nuevo.
  3. Acepta los términos y condiciones.

4. Registro de Web App & SDK

Para conectar Google AI Studio con Firebase, necesitamos el **SDK Configuration Code**. Este bloque de código contiene las llaves maestras de conexión.

¡Ojo! Copia y guarda este código; lo inyectaremos en el prompt del agente más adelante.

5. Métodos de Acceso

Habilita las puertas de entrada en la pestaña de Authentication:

  • Email/Password: El método clásico y obligatorio para control total.
  • Google Sign-in: La forma más rápida de convertir visitas en usuarios registrados.

6. Personalización de Correos

Configura los emails de bienvenida y recuperación. Si no vinculas un dominio profesional, llegarán a la carpeta de SPAM, pero funcionarán perfectamente para tests.

7. El Prompt Maestro de Auth

Inyecta esta lógica en Google AI Studio para crear el sistema de login automáticamente:

"Antes de usar el Drive, el usuario debe registrarse o loguearse con Firebase.
Usa este SDK: [TUPUM_SDK_AQUÍ].
Campos: Foto de perfil, Nombre, Email, Password. 
Si ya existe, muéstrale 'User already exists, sign in instead'."

8. Verificación de Email y Password Reset

No permitas el acceso sin validación. Implementa un flujo que detenga al usuario hasta que haga clic en el link de confirmación enviado a su correo.

Tip: Usa el comando Firebase.auth.currentUser.sendEmailVerification() para automatizarlo.

9. Firestore Database: El Inventario

Configura una colección /users. Cada documento debe tener el nombre del UID (User ID) único para asegurar que un usuario nunca vea los datos de otro.

10. Reglas de Seguridad Pro

Las reglas de Firebase son las que realmente cierran el casillero. Usa:

match /users/{userId} {
  allow read, write: if request.auth.uid == userId;
}

11. Firebase Storage: La Nube Real

Storage guarda los archivos pesados (PDFs, Imágenes, Audios). Por cada archivo guardado, el sistema debe salvar el Storage Path en Firestore para que el UI pueda localizarlo más tarde.

12. Reglas de Acceso a Archivos

Al igual que en la DB, Storage debe restringir el acceso. Crea una carpeta /user_uploads/{uid}/ donde solo el dueño tenga permisos de lectura y escritura.

13. Descargas y Sincronización

Para descargar, la app solicita un Download URL temporal basado en el path guardado. Esto abre el archivo en una pestaña nueva para que el usuario pueda guardarlo en su disco local.

14. Bonus: Cálculo de Quota (5GB)

Firebase te da 5GB gratis. Tutorial avanzado: Suma el campo size de todos los documentos en /users/{uid}/files/ para mostrar una barra de progreso real en el dashboard.

Fórmula: (Total Bytes Sumados / 5 * 1024^3) * 100
