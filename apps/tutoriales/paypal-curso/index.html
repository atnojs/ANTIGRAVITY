YouTube Learner: Pagos con PayPal

Transforma tus ideas en beneficios reales.

Este tutorial te enseña a implementar un sistema de monetización completo utilizando PayPal y Make.com. Aprenderás a gestionar suscripciones, créditos de uso y automatizaciones complejas sin escribir código de backend tradicional.

Ver Tutorial en YouTube

1. El fin del desarrollo gratuito

Lanzar un negocio SaaS hoy no requiere miles de dólares. Gracias a la IA y herramientas de automatización, puedes construir sistemas de cobro recurrentes que escalen automáticamente, permitiéndote monetizar tus herramientas de IA desde el primer día.

2. App Setup: El Cimiento

Iniciamos con una base sólida en Google AI Studio conectada a Firebase. Es vital tener la autenticación y la base de datos Firestore listas antes de introducir la lógica de pagos.

Recordatorio: Necesitas tener el `iDToken` de Firebase configurado para validar peticiones seguras desde Make.com.

3. El Sistema de Créditos Gratuitos

Atrae usuarios con un modelo Freemium. Cada nuevo registro recibe automáticamente 10 créditos para probar la herramienta.

// Lógica Firestore al añadir video:
videos_used = videos_used + 1;
if (videos_used >= 10 && plan == "free") {
   disable_access();
}

4. Diseño de la Página de Planes

Crea una interfaz clara de precios: Plus ($20) y Premium ($40). El diseño debe invitar al upgrade una vez se agoten los créditos gratuitos.

5. PayPal Sandbox Explorer

Nunca pruebes con dinero real al inicio. Usa el **Sandbox** de PayPal para simular compras exitosas y fallidas utilizando cuentas ficticias de "Vendedor" y "Comprador".

  1. Activa el modo Sandbox en el Developer Dashboard.
  2. Obtén tu **Client ID** y **Secret Key**.
  3. Usa los WebHook Simulators para probar escenarios de error.

6. Creación de Productos & Planes

En tu panel de PayPal Sandbox, define los productos. Cada plan genera un Plan ID único (ej: P-LLX...) que usaremos para indicar a la API cuánto cobrar.

7. Make.com: Generador de Links

El primer escenario de Make recibe el `iDToken` del usuario, valida su identidad contra Firebase y solicita a PayPal un link de pago único. Este link se devuelve al front-end para la redirección.

8. Sincronización con Firestore

Cuando el pago se completa, PayPal dispara un WebHook. Make.com captura este evento y actualiza el campo plan y status en el documento del usuario en tiempo real.

9. Listeners en AI Studio

Configuramos el front-end para que "escuche" cambios en el documento de usuario. En cuanto status pasa a active, se desbloquean las funciones de análisis premium automáticamente.

10. Loader de Verificación

Implementa un estado UX intermedio. Mientras Make procesa el pago, muestra: "Verificando tu pago... esto puede tardar unos segundos". Esto evita confusiones si la base de datos tarda un instante en actualizarse.

11. El Flujo de Cancelación

Todo buen sistema de pagos debe dejar ir al usuario con elegancia. Implementa un pop-up de confirmación pidiendo el motivo de la baja (feedback valioso).

12. Make.com: Automatizando Bajas

Un segundo escenario de Make llama al endpoint /billing/subscriptions/{id}/cancel de PayPal para detener el cobro recurrente de forma inmediata.

13. Gestión de Estados: Cancelado vs Expirado

Diferencia entre:

  • Cancelled: El usuario detuvo el cobro pero aún tiene acceso hasta el fin de mes.
  • Expired/Failed: Se revoca el acceso y el usuario vuelve al plan gratuito.

14. Checklist para el GoLive

Antes del lanzamiento real:

  1. Cambia el entorno Sandbox a **Live** en PayPal y Make.
  2. Actualiza los **Price/Plan IDs** con versiones reales.
  3. Configura los Webhooks de producción.
  4. Vuelve a testear cada flujo.

15. Conclusión: Monetización Inteligente

Has construido un motor de ingresos SaaS. La integración de PayPal con automatizaciones sin código es el atajo más rápido para convertir tus habilidades de IA en una fuente de ingresos real y escalable.
