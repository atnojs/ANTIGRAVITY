Booking System con Stripe y Firebase

Domina los pagos digitales y la infraestructura No-Code/Low-Code.

Este tutorial te guiará en la creación de una plataforma de reservas funcional: desde el diseño con Gemini 3 hasta la integración real de pasarelas de pago con Stripe.

Ver Tutorial en YouTube

1. El fin de los negocios tradicionales

Google AI Studio junto a Gemini 3 está transformando el desarrollo SaaS. Lo que antes requería un equipo de frontend senior, ahora se puede lograr integrando herramientas como Firebase para el backend y Stripe para la monetización en cuestión de horas.

Objetivo: Crear un dashboard de reservas donde los clientes puedan pagar por slots de consultoría y recibir confirmaciones automáticas.

2. Prompt Inicial: El Diseño Minimalista

Iniciamos en Google AI Studio pidiendo la estructura visual. Busca un diseño minimalista con colores pastel y blancos para una experiencia de usuario suave.

Prompt Pro: "Crea un dashboard de reservas de sesiones de IA con diseño minimalista blanco y pastel. El usuario debe autenticarse, ver slots disponibles según mi zona horaria y poder reservar sesiones de 1h o 2h con un botón de Stripe."

3. Setup Backend con Firebase

Firebase será nuestro motor. Configura los pilares fundamentales:

  1. Crea un proyecto en la consola de Firebase.
  2. Activa Authentication: Habilita los métodos Email/Password y Google Sign-in.
  3. Crea una Web App para obtener tu configuración SDK.

4. Estructura de Firestore

Nuestra base de datos necesita dos colecciones clave para funcionar:

  • /settings/availability: Documento con tu horario semanal mappreado (ej: EST timezone).
  • /global_bookings: Colección para bloquear slots ocupados y evitar "overbooking".
Importante: La aplicación debe leer de ambas para mostrar solo los slots realmente disponibles al cliente.

5. Reglas de Seguridad

Protege los datos de tus usuarios. Las reglas de Firebase deben asegurar que:

// Solo el dueño lee el schedule
match /settings/availability {
  allow read: if true;
  allow write: if isAdmin();
}
// Los usuarios solo ven y editan sus propias reservas
match /users/{userId}/bookings/{bookingId} {
  allow read, write: if request.auth.uid == userId;
}

6. Conexión SDK

Copia el código de configuración de Firebase e inyéctalo en el prompt de Gemini 3. Esto le permite al agente escribir funciones reales de login y registro vinculadas a tu base de datos.

7. Sincronización de Disponibilidad

El sistema debe detectar la zona horaria del usuario automáticamente. Si tu agenda está en EST y el cliente en España, el calendario debe mostrar las horas ajustadas sin intervención manual.

8. Setup de Stripe Sandbox

Antes de cobrar dinero real, usa el entorno de pruebas (Sandbox):

  1. Crea productos en Stripe: "AI Session 1h" y "AI Session 2h".
  2. Elige One-time payment (pago único).
  3. Copia los Price IDs (comienzan por price_...).

9. Extensión Firebase-Stripe

Instala la extensión "Run Payments with Stripe" desde el Hub de extensiones de Firebase. Esto automatiza la creación de sesiones de pago y el manejo de suscripciones.

Configuración: Pega tu Secret Key de Stripe en los ajustes de la extensión para vincular ambos servicios.

10. Webhooks: Comunicación de Retorno

Para que Firebase sepa si un pago tuvo éxito, Stripe debe enviar un "mensaje" de vuelta.

  1. Copia el Endpoint URL desde las Funciones de Firebase (handleWebhookEvents).
  2. Suscrito a eventos: checkout.session.completed y payment_intent.succeeded.
  3. Configura el Webhook Secret en Firebase para validar que los mensajes vienen de Stripe.

11. Flujo de Pago Real

Cuando el usuario pulsa "Pagar", invocamos la función createCheckoutSession pasando:

  • El Price ID seleccionado.
  • La URL de éxito (/success) y fracaso (/failure).
  • El User ID para saber a quién pertenece el pago.

12. Validación y Rutas Post-Pago

Tras el pago exitoso, Stripe añade el ID de transacción a Firestore. El tutorial concluye enseñando a Gemini a gestionar la ruta /success para:

  1. Mostrar un mensaje de confirmación al usuario.
  2. Vincular el Payment ID a la reserva original en la base de datos.
  3. Mostrar los detalles del ticket (precio, fecha, estado) en el dashboard.

13. Conclusión: El desarrollador Full-Stack AI

Has pasado de tener una idea a un sistema de pagos funcional. La clave está en delegar la lógica repetitiva a la IA y enfocarte en la orquestación de servicios premium como Stripe y Firebase.
