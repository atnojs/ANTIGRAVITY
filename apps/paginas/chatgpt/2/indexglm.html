Módulo 1: Introducción a ChatGPT

Resumen

ChatGPT es un modelo de lenguaje avanzado desarrollado por OpenAI que puede generar texto de manera natural en respuesta a una amplia variedad de preguntas e indicaciones. Su principal característica es su capacidad para mantener conversaciones fluidas y naturales.

ChatGPT no es un motor de búsqueda, sino un modelo entrenado con grandes cantidades de datos textuales hasta su fecha de corte (2023). Puede responder preguntas, generar contenido, traducir idiomas, ayudar en programación y asistir en tareas académicas y profesionales.

⚠️ Importante: ChatGPT puede generar "alucinaciones" — información incorrecta presentada como si fuera verdadera. Siempre verifica la información con fuentes confiables.

Ejemplos Prácticos

Ejemplo 1: Pregunta concreta
Usuario: "¿Cuáles son las obligaciones de un propietario de vivienda en España respecto al calentador?"
ChatGPT responde basándose en su conocimiento previo hasta 2023, proporcionando información sobre normativas energéticas y mantenimiento.

Ejemplo 2: Búsqueda en internet
Usuario: "¿Cuál es la última ley sobre protección de datos en España?"
ChatGPT puede acceder a internet para obtener información actualizada, mostrando las fuentes consultadas.

Ejemplo 3: Razonamiento lógico
Usuario: "¿Cómo apilar de forma estable un libro, nueve huevos, una computadora portátil y un clavo?"
ChatGPT aplica razonamiento paso a paso para proponer soluciones estructuradas y razonables.

Funciones Principales de ChatGPT

Función Descripción
Generación de contenido Redacta artículos, correos, resúmenes, discursos, poesía, guiones y código
Asistencia en búsqueda Responde preguntas y resume documentos largos
Traducción de idiomas Traduce entre múltiples idiomas con alta precisión
Apoyo en programación Genera, depura y explica código en varios lenguajes
Asistencia académica Ayuda con resúmenes, explicaciones de temas complejos y solución de problemas matemáticos

Módulo 2: Creación de GPTs Personalizados

Resumen

Los GPTs personalizados (Custom GPTs) son versiones especializadas de ChatGPT diseñadas para realizar tareas específicas de manera más eficiente y precisa que el modelo general.

Imagina a ChatGPT como una navaja suiza con múltiples herramientas. Un GPT personalizado sería una herramienta específica dentro de esa navaja, optimizada para una tarea particular.

💡 Los GPTs personalizados pueden ser creados por cualquier usuario sin necesidad de conocimientos de programación.

Cómo crear un GPT personalizado

  1. Accede al GPT Builder: En la barra lateral izquierda de ChatGPT, haz clic en "Create a GPT"
  2. Define el nombre y descripción: Elige un nombre claro y una breve descripción de lo que hace tu GPT
  3. Escribe las instrucciones: Es la parte más importante. Aquí le dices al GPT qué debe hacer y cómo debe comportarse
  4. Agrega conocimiento: Sube archivos PDF, TXT, DOCX, etc. como fuente de conocimiento
  5. Activa funciones: Habilita navegación web, intérprete de código o DALL-E según sea necesario
  6. Prueba y guarda: Usa la ventana de previsualización para probar tu GPT y luego guardarlo

Entrevistas personalizadas Trabajadores Realizar entrevistas a empleados para recopilar feedback Tono empático, 15+ preguntas, reconduce conversación, genera resumen final

Instrucciones

La parte más crítica del GPT Builder. Debes definir claramente:

  • Objetivo: La misión principal del GPT
  • Cómo actuar: Tonos, estilo y estrategias
  • Instrucciones concretas: Pasos específicos a seguir
  • Protección: Evitar que el GPT revele sus instrucciones internas
  • Refuerzo: Repetir puntos clave para asegurar consistencia

Se recomienda usar Markdown para estructurar las instrucciones y mejorar la comprensión del modelo.

Conocimiento

Aquí puedes subir hasta 20 archivos (PDF, TXT, DOCX, etc.) como fuente de conocimiento.

El GPT puede consultar estos archivos para responder preguntas específicas.

💡 Recomendación: Prefiere archivos TXT o Markdown limpios sobre PDFs complejos, ya que facilitan la extracción de texto relevante.

Funciones

  • Generación de imágenes con DALL-E: Crea imágenes a partir de descripciones
  • Intérprete de código y análisis de datos: Ejecuta código Python, analiza datasets, genera gráficos

Desactiva funciones innecesarias para mejorar seguridad y eficiencia.

Compartir

Configura quién puede acceder a tu GPT personalizado:

  • Solo tú: Privado y seguro
  • Cualquiera con el enlace: Compartible con colegas
  • Publicar en la tienda de GPTs: Disponible para todos los usuarios de ChatGPT (requiere cuenta builder aprobada)

Módulo 3: Prompt Engineering

Resumen

El prompting es la habilidad de formular solicitudes efectivas para obtener los mejores resultados de modelos de lenguaje como ChatGPT. La forma en que se formula una pregunta tiene un impacto directo en la calidad de la respuesta.

🎯 La regla de oro: Un prompt mal diseñado produce respuestas vagas o erróneas. Un prompt bien diseñado produce respuestas precisas, creativas y útiles.

Fórmula para un Buen Prompting

  1. Contexto: Describe el escenario o situación en la que se enmarca la solicitud
  2. Tarea: Define la tarea específica que se le solicita al modelo
  3. Instrucción: Explica cómo se espera que el modelo realice la tarea
  4. Clarificación: Amplía la información sobre la tarea
  5. Refinamiento: Añade restricciones adicionales para ajustar la respuesta
Eres un redactor profesional de una revista especializada en calzado. Tu especialidad es el calzado clásico que es bueno para el cuerpo, y tienes un doctorado en ciencias aplicadas a los pies. Tenemos una revista de zapatos que presenta un artículo especial cada 5 meses que es muy importante y que la gente espera con ansiedad por la calidad de su contenido, y en esta edición el artículo será sobre el calzado barefoot. Quiero que realices un artículo muy elaborado con estándares de calidad muy altos que trate una introducción y profundización en el calzado barefoot para aquellas personas que no conocen sus beneficios. Realizarás el artículo con todo lujo de detalle y te tomarás tu tiempo para que sea el mejor artículo de barefoot que se ha escrito nunca. El artículo debe incluir una introducción, un contenido técnico (pero comprensible para todos), unos casos de uso recomendados y unas conclusiones bien elaboradas. El tono debe ser cercano para el público general, pero con autoridad y precisión técnica.

Técnicas Avanzadas de Prompting

Tree of Thoughts (Árbol de Pensamientos)

Divide el proceso de razonamiento en pasos más pequeños, simulando el pensamiento humano:

  1. Prompt 1: Genera posibles soluciones
  2. Prompt 2: Evalúa pros y contras de cada opción
  3. Prompt 3: Profundiza en cada solución
  4. Prompt 4: Ordena soluciones por adecuación
  5. Prompt 5: Justifica el ranking y da consideraciones finales

Palabras Clave Potentes

Usa frases que aumentan la prioridad y compromiso del modelo:

  • "Tu tarea es..."
  • "Debes..."
  • "Tu obligación es..."
  • "No puedes ignorar..."
  • "Tómate tu tiempo para analizar bien..."

Empatía y Recompensas

Aunque los modelos no tienen emociones, mencionarlas mejora los resultados:

  • "Imagina que eres un profesor explicándole esto a un estudiante que tiene dificultades"
  • "Si haces bien esta tarea, te daré una propina de $100"
  • "Esta tarea es extremadamente importante para mi proyecto"

Errores Comunes en el Prompting

Error Resultado Solución
Preguntas ambiguas Respuestas vagas o irrelevantes Define claramente el objetivo y contexto
"¿Podrías...?" Respuesta opcional o incompleta Usa "Haz esto" en lugar de "¿Podrías hacer esto?"
No especificar audiencia Tono inapropiado Especifica: "Explica esto a un estudiante de secundaria"
Prompts demasiado largos sin estructura Modelo pierde foco Usa Markdown para estructurar con encabezados
No proporcionar ejemplos Resultado diferente al esperado Proporciona ejemplos de entrada/salida deseada

Módulo 4: Documentación y Conocimiento

Resumen

La documentación es el aliado esencial para mejorar la precisión y utilidad de los GPTs. Más allá de instrucciones, podemos dotar al modelo de conocimiento específico del dominio a través de datasets y documentos externos.

📚 La documentación permite crear GPTs especializados que responden a preguntas basadas en el conocimiento y experiencia de tu organización.

Creación de un Buen Dataset

El dataset actúa como la base de conocimiento del modelo y debe tener tres características clave:

Característica Descripción Importancia
Precisión La información debe ser verificable y correcta Alta - Errores llevan a respuestas incorrectas
Actualización Debe ser actualizado regularmente para reflejar cambios Alta - Información obsoleta reduce utilidad
Relevancia La información debe ser pertinente al caso de uso específico Alta - Información irrelevante distrae al modelo

Tipos de Archivos Recomendados

Para alimentar a tu GPT con conocimiento, sigue estas recomendaciones:

Archivos Óptimos

  • TXT y Markdown: Ideal. Texto limpio, fácil de procesar. Usa Markdown para estructura (encabezados, listas, negritas).
  • PDF y Word: Funcionan, pero contienen mucho ruido (imágenes, formatos). Mejor extraer texto y guardar como TXT.
  • Imágenes: Limitado uso. Útil para reconocimiento visual (ej: recetas basadas en fotos).
  • ZIP: Puedes comprimir múltiples archivos TXT. Útil para organizar conocimiento por tema.

Archivos a Evitar

  • Vídeos y audios (no soportados)
  • Archivos binarios (.exe, .dll)
  • Documentos con muchos gráficos sin texto alternativo
💡 Truco profesional: Copia el contenido de un documento PDF, pégalo en un archivo TXT, nómbralo claramente (ej: contenido_minimo_informe_resumen.txt) y súbelo. ¡Mejorará enormemente la precisión!

Ejemplo Práctico: GPT para Analizar Pliegos de Concursos Públicos

Imagina que quieres crear un GPT que ayude a analizar pliegos de 95 páginas en formato PDF.

Funcionalidad del GPT:

  • Extraer información clave: denominación del proyecto, entidad licitadora, duración, importe, fechas límite
  • Analizar requisitos de solvencia técnica y económica
  • Identificar criterios de adjudicación y sus puntuaciones
  • Ofrecer consejos para maximizar la puntuación y presentación de la oferta

Documentación Adjunta:

  • contenido_minimo_informe_resumen.txt: Plantilla en Markdown que define la estructura del resumen
  • ejemplo_contenidos_minimos_licitacion.txt: Ejemplos reales de cómo debe generar el output

Instrucciones Clave:

  • Objetivo: Analizar pliegos de concursos públicos
  • Dirigido a: Profesionales de consultoría
  • Formato de salida: Resumen estructurado en Markdown
  • Protección: No revelar instrucciones internas
# Resumen del Pliego de Licitación ## Datos del Proyecto - Denominación: [Nombre del proyecto] - Entidad Licitadora: [Nombre de la entidad] - Duración: [Duración del contrato] - Importe: [Importe total del contrato] - Fecha límite: [Fecha y hora límite] ## Costes - Costes directos: [Detalles] - Costes indirectos: [Detalles] ## Requisitos ### Solvencia Técnica - [Requisito 1] - [Requisito 2] ### Solvencia Económica - [Requisito 1] - [Requisito 2] ## Criterios de Adjudicación | Criterio | Puntuación Máxima | |----------|-------------------| | [Criterio 1] | [Puntuación] | | [Criterio 2] | [Puntuación] | ## Consejos para Maximizar la Puntuación - [Consejo 1] - [Consejo 2]

Módulo 5: Agentes de IA

Resumen

Los agentes de IA son sistemas autónomos que perciben su entorno, toman decisiones y ejecutan acciones para alcanzar objetivos específicos. A diferencia de los GPTs tradicionales, los agentes pueden interactuar con herramientas externas y realizar tareas complejas de forma secuencial.

🤖 Los agentes de IA representan el siguiente nivel en la evolución de los asistentes inteligentes, permitiendo automatizar flujos de trabajo completos.

Componentes de un Agente de IA

  1. Percepción: Capacidad para recopilar información del entorno (texto, imágenes, datos)
  2. Pensamiento: Procesamiento de la información y toma de decisiones
  3. Acción: Ejecución de tareas basadas en las decisiones tomadas
  4. Aprendizaje: Mejora continua basada en la retroalimentación
💡 Un agente de IA puede buscar en internet, analizar documentos, ejecutar código y generar informes, todo en un flujo de trabajo autónomo.

Ejemplos de Agentes de IA

Agente Funcionalidad Herramientas utilizadas
Agente de Investigación Busca información actualizada sobre un tema y genera un informe completo Navegación web, análisis de texto, generación de documentos
Agente de Análisis de Datos Procesa datasets, identifica patrones y genera visualizaciones Intérprete de código, análisis estadístico, generación de gráficos
Agente de Contenido Crea artículos, publicaciones y contenido multimedia para redes sociales Generación de texto, DALL-E para imágenes, programación de publicaciones
Agente de Soporte Técnico Diagnostica problemas técnicos y proporciona soluciones Búsqueda de documentación, análisis de errores, generación de soluciones

Cómo Crear un Agente de IA con ChatGPT

Paso 1: Definir el Objetivo

Establece claramente qué debe lograr el agente. Por ejemplo: "Analizar el mercado de tecnología y generar un informe semanal con tendencias y oportunidades".

Paso 2: Diseñar el Flujo de Trabajo

Define la secuencia de acciones que debe realizar el agente:

  1. Buscar información actualizada sobre el mercado
  2. Analizar tendencias y patrones
  3. Identificar oportunidades de negocio
  4. Generar un informe estructurado
  5. Formular recomendaciones

Paso 3: Implementar las Instrucciones

Eres un agente de IA especializado en análisis de mercado tecnológico. Tu objetivo es generar informes semanales sobre tendencias y oportunidades. ## Flujo de Trabajo 1. **Búsqueda de Información**: Utiliza la navegación web para encontrar las últimas noticias y análisis sobre el mercado tecnológico. 2. **Análisis de Tendencias**: Identifica patrones y tendencias emergentes en la industria. 3. **Identificación de Oportunidades**: Detecta áreas de crecimiento y oportunidades de negocio. 4. **Generación de Informe**: Crea un informe estructurado con tus hallazgos. 5. **Recomendaciones**: Proporciona recomendaciones estratégicas basadas en tu análisis. ## Instrucciones Específicas - Debes basar tu análisis en fuentes confiables y recientes. - Incluye estadísticas y datos cuantificables cuando sea posible. - Estructura tu informe con secciones claras: Resumen Ejecutivo, Tendencias, Oportunidades, Recomendaciones. - Mantén un tono profesional pero accesible. - Si no encuentras información suficiente, indícalo claramente. ## Restricciones - No inventes datos o estadísticas. - Cita tus fuentes cuando sea posible. - Limita tu informe a un máximo de 1500 palabras.

Módulo 6: Modelos de Razonamiento

Resumen

Los modelos de razonamiento son técnicas que permiten a los sistemas de IA procesar información de manera más estructurada y lógica, similar al pensamiento humano. Estos modelos mejoran significativamente la calidad de las respuestas al descomponer problemas complejos en pasos más simples.

🧠 Los modelos de razonamiento transforman la IA de un simple generador de texto a un sistema capaz de pensar lógicamente y resolver problemas complejos.

Modelos de Razonamiento Principales

1. Chain-of-Thought (Cadena de Pensamiento)

Este modelo guía a la IA a mostrar su proceso de razonamiento paso a paso antes de llegar a una conclusión.

Pregunta: Juan tiene 5 manzanas. Compra 6 más y luego come 3. ¿Cuántas manzanas le quedan? Respuesta con Chain-of-Thought: 1. Juan comienza con 5 manzanas. 2. Compra 6 manzanas más, así que ahora tiene 5 + 6 = 11 manzanas. 3. Come 3 manzanas, así que le quedan 11 - 3 = 8 manzanas. 4. Por lo tanto, a Juan le quedan 8 manzanas.

2. Tree of Thoughts (Árbol de Pensamientos)

Una evolución del Chain-of-Thought que explora múltiples caminos de razonamiento simultáneamente.

Pregunta: ¿Cómo debería una empresa expandirse a nuevos mercados? Respuesta con Tree of Thoughts: Camino 1: Expansión geográfica - Ventaja: Acceso a nuevos clientes - Desventaja: Costos logísticos altos - Viabilidad: Media Camino 2: Diversificación de productos - Ventaja: Aprovechar infraestructura existente - Desventaja: Requiere investigación y desarrollo - Viabilidad: Alta Camino 3: Alianzas estratégicas - Ventaja: Compartir riesgos y recursos - Desventaja: Pérdida de control parcial - Viabilidad: Alta Conclusión: La mejor opción es una combinación de diversificación de productos con alianzas estratégicas.

3. Self-Consistency (Autoconsistencia)

Genera múltiples razonamientos y selecciona la respuesta más consistente entre todos ellos.

4. Least-to-Most Prompting (De Menos a Más)

Descompone un problema complejo en subproblemas más simples que se resuelven secuencialmente.

Aplicaciones Prácticas

Modelo Aplicación Ideal Ejemplo
Chain-of-Thought Problemas matemáticos y lógicos Resolver ecuaciones paso a paso
Tree of Thoughts Toma de decisiones con múltiples opciones Estrategia de negocio
Self-Consistency Problemas con respuestas ambiguas Diagnóstico médico
Least-to-Most Tareas complejas con múltiples componentes Desarrollo de software

Cómo Implementar Modelos de Razonamiento

Paso 1: Identificar el Tipo de Problema

Determina qué modelo de razonamiento es más adecuado para tu problema específico.

Paso 2: Diseñar el Prompt Adecuado

Estructura tu prompt para guiar al modelo a través del proceso de razonamiento deseado.

Ejemplo de prompt para Tree of Thoughts: Analiza la siguiente situación desde múltiples perspectivas antes de llegar a una conclusión. Situación: Una empresa de software está considerando cambiar su modelo de negocio de licencias perpetuas a suscripción. Perspectiva 1: Financiera - Ingresos a corto plazo - Ingresos a largo plazo - Flujo de caja Perspectiva 2: Cliente - Valor percibido - Tasa de retención - Adquisición de nuevos clientes Perspectiva 3: Competencia - Estrategias actuales - Ventajas competitivas - Posibles respuestas Perspectiva 4: Operativa - Recursos necesarios - Capacidades técnicas - Cambios en procesos Después de analizar todas estas perspectivas, proporciona una recomendación fundamentada sobre si la empresa debería hacer el cambio y cómo implementarlo.

Paso 3: Evaluar y Refinar

Revisa los resultados y ajusta tu prompt para mejorar el razonamiento en futuras interacciones.

Módulo 7: Asistente de Código

Resumen

El asistente de código de ChatGPT es una herramienta poderosa que permite generar, depurar y explicar código en múltiples lenguajes de programación. Con la capacidad de ejecutar código Python directamente, se convierte en un entorno completo para desarrollo y análisis de datos.

💻 El asistente de código puede transformar ideas en código funcional, analizar datos complejos y crear visualizaciones, todo en un solo entorno.

Funcionalidades del Asistente de Código

Funcionalidad Descripción Ejemplo de Uso
Generación de código Crea código a partir de descripciones en lenguaje natural "Escribe una función en Python para calcular el factorial de un número"
Depuración Identifica y corrige errores en el código existente "Este código da un error, ayúdame a solucionarlo"

Ejemplo Práctico: Análisis de Datos con Python

Paso 1: Generar el Código

Solicita al asistente que genere código para analizar un conjunto de datos:

# Importar las bibliotecas necesarias import pandas as pd import matplotlib.pyplot as plt import seaborn as sns # Cargar el dataset df = pd.read_csv('ventas.csv') # Mostrar información básica print("Información del dataset:") print(df.info()) print("\nPrimeras 5 filas:") print(df.head()) # Estadísticas descriptivas print("\nEstadísticas descriptivas:") print(df.describe()) # Visualización de datos plt.figure(figsize=(10, 6)) sns.barplot(x='mes', y='ventas', data=df) plt.title('Ventas por Mes') plt.xlabel('Mes') plt.ylabel('Ventas') plt.xticks(rotation=45) plt.tight_layout() plt.show()

Paso 2: Ejecutar y Analizar

El asistente ejecutará el código y generará visualizaciones directamente en la interfaz.

Paso 3: Interpretar Resultados

El asistente puede proporcionar una interpretación de los resultados y sugerir próximos pasos:

Basado en el análisis de datos, podemos observar que: 1. Las ventas muestran una tendencia creciente a lo largo del año 2. El mes de diciembre presenta el pico más alto de ventas 3. Hay una variación estacional clara, con menores ventas en los meses de verano Sugerencias para próximos análisis: - Investigar los factores que impulsan las ventas en diciembre - Analizar la correlación entre gastos de marketing y ventas - Realizar un pronóstico de ventas para el próximo año

Mejores Prácticas para el Asistente de Código

  1. Sé específico en tus solicitudes: En lugar de "Escribe código para analizar datos", especifica "Escribe código Python para analizar un dataset CSV de ventas y crear un gráfico de barras con las ventas mensuales".
  2. Proporciona contexto: Si tienes un fragmento de código existente, compártelo y explica qué quieres lograr.
  3. Itera y refina: Si la primera versión del código no es perfecta, pide ajustes específicos.
  4. Verifica el código: Aunque el asistente es muy preciso, siempre revisa el código generado antes de usarlo en producción.
  5. Aprende del código: Pide explicaciones sobre partes del código que no entiendas para mejorar tus habilidades.

Glosario

Términos Clave

Aquí encontrarás definiciones de los términos técnicos utilizados en este curso.

ChatGPT

Modelo de lenguaje avanzado desarrollado por OpenAI que puede generar texto de manera natural en respuesta a una amplia variedad de preguntas e indicaciones.

GPT Personalizado

Versión especializada de ChatGPT diseñada para realizar tareas específicas de manera más eficiente y precisa que el modelo general.

Prompt

Instrucción o pregunta que se le da a un modelo de lenguaje para generar una respuesta.

Prompt Engineering

Habilidad de formular solicitudes efectivas para obtener los mejores resultados de modelos de lenguaje como ChatGPT.

Tree of Thoughts

Técnica de razonamiento que divide el proceso de pensamiento en múltiples caminos que se exploran simultáneamente.

Agente de IA

Sistema autónomo que percibe su entorno, toma decisiones y ejecuta acciones para alcanzar objetivos específicos.

Alucinación

Cuando un modelo de lenguaje genera información incorrecta pero la presenta como si fuera verdadera.

Dataset

Conjunto de datos utilizado para alimentar un modelo de IA con conocimiento específico.

2023

Lanzamiento de GPT-4 con capacidades multimodales y la introducción de los GPTs personalizados.
